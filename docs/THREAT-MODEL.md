# Модель угроз и дисклеймер

## Против чего это

- DPI/ТСПУ, режущий известные VPN-протоколы по сигнатуре и поведению.
- Блокировки по SNI, по паттерну хендшейков, фингерпринт VLESS-Reality.
- Гео-ограничения сервисов по IP/региону (через split-tunnel и опц. WARP-выход).

Подход - разнообразие: несколько протоколов (QUIC/TCP), маскировка под обычный HTTPS-сайт,
firefox-отпечаток, XHTTP-мультиплекс, опционально CDN-фронтинг (прячет IP/ASN сервера).
Если один путь блокируют, клиент идёт по другому.

## Чего это НЕ делает

- Не даёт анонимность. Это инструмент доступа/обхода блокировок, не Tor. Выходной IP -
  это IP твоего сервера; провайдер выхода и хостер видят трафик на своём уровне.
- Не гарантирует обход в любой сети. DPI у конкретного провайдера может ломать конкретный
  протокол (например троттлить UDP/QUIC) - на это и нужны запасные узлы.
- Не защищает от компрометации клиента/сервера. Безопасность сервера - на тебе (ключи,
  обновления, доступы).
- CDN-фронтинг и WARP не дают 100% стабильности: узкое место - сеть на стороне блокировок,
  а не план хостинга.

## Юридическое

Самостоятельный VPN для личного/семейного доступа в открытый интернет. Ты сам отвечаешь за
законность использования в своей юрисдикции и за соблюдение правил своего хостера. Проект -
код и инструкции, без серверов, без чьих-либо данных. Лицензия MIT, "как есть", без гарантий.

## Гигиена секретов

- Все ключи/пароли генерируются НА твоём сервере при установке и лежат только в `.env`
  (chmod 600) и конфигах сервисов. В репозитории секретов нет.
- Не публикуй `.env`, `db.sqlite`, приватные ключи, реальные ссылки подписок.
- `.gitignore` уже исключает это; не коммить их и при доработке.
