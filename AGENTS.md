# AGENTS.md - установка через AI-агента

Этот файл - инструкция для AI coding-агента (Claude Code, Cursor, Aider или совместимого),
который ставит VPN-узел и бота на сервер пользователя. Агент работает либо запущенный
прямо НА сервере, либо локально с SSH-доступом к серверу.

Цель: пользователь даёт домен и доступ, агент проходит установку и отдаёт готовые ссылки
подписки. Скрипты делают тяжёлую работу и идемпотентны - повторный запуск безопасен.

## Что нужно от пользователя (prerequisites)

1. **VPS** - Ubuntu 22.04+ / Debian 12+, root-доступ. Чистый, желательно отдельный под VPN.
2. **Домен** - и DNS **A-запись** на IP сервера. Поддомен годится (`vpn.example.com`).
   Проверка: `dig +short ДОМЕН` должен вернуть IP сервера.
3. **SSH** - если агент работает не на сервере: `ssh root@IP` должен пускать (ключ в
   `authorized_keys`). Проверка: `ssh root@IP 'echo ok'`.
4. **(опц.) Cloudflare** - для узла XHTTP+CDN: домен на Cloudflare, A-запись на сервер в
   режиме Proxied, SSL/TLS mode = Full. Без этого узел XHTTP+CDN просто не добавится.

## Порядок действий агента

### Шаг 1. Узел VPN
1. Подключись к серверу (или открой терминал на нём).
2. Склонируй репозиторий в `/root/vpnbanana`.
3. Спроси у пользователя домен (и убедись, что A-запись уже указывает на сервер - `dig`).
4. Запусти: `DOMAIN=<домен> bash /root/vpnbanana/server/install.sh`
   Нестандартные порты - через env (`HY_MAIN_PORT=`, `REALITY_PORT=` и т.д.), если 443
   занят другим сервисом на этом боксе.
5. Скрипт сгенерит все ключи/пароли, поднимет Hysteria2 + Xray + nginx, выпустит TLS-серт,
   откроет порты в ufw и напечатает: открытые порты, ссылку личной подписки, ссылку на
   список обхода. Покажи это пользователю.

### Шаг 2. Бот выдачи подписок
1. Попроси у пользователя токен бота (@BotFather) и его Telegram id (@userinfobot).
2. Запусти: `BOT_TOKEN=<токен> ADMIN_ID=<id> bash /root/vpnbanana/bot/install.sh`
3. Скрипт поднимет backend+бота+учёт, переключит Hysteria на пер-юзер http-auth,
   проверит здоровье `/auth`. Скажи пользователю открыть бота и нажать `/start`.

### Шаг 3 (опц.). Вторая локация / WARP
- WARP-выход для гео-AI (Gemini/OpenAI): `bash /root/vpnbanana/server/warp-egress.sh`.
- Вторая локация: на втором VPS `PL_HOST=<домен2> bash server/add-location.sh`, затем
  вставь напечатанный блок `PL_*` в `/root/vpnbanana/.env` основного и перезапусти бота.

## Правила для агента

- Не коммить и не пересылай содержимое `.env`, `db.sqlite`, ключей - это секреты сервера.
- Если порт занят (общий сервер) - подбери свободный и передай его через env, не ломай
  существующие сервисы на боксе.
- При сбое читай `journalctl -u vpnbanana-backend -n50`, `systemctl status xray hysteria-server`,
  `nginx -t`. Скрипты идемпотентны - чини причину и запускай повторно.
- Готовые тексты задач - в `prompts/`.
